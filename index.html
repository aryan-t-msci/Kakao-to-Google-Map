Kakao-side view · setup

You don't need this for spot-checks. The coordinate, the Google satellite, and all the links work without it. This switches on the Kakao satellite next to Google and name/address search. It's a one-time setup and it only works when this page is opened from a real web address (a hosted link or localhost) — not from a double-clicked file.
1
Get a free Kakao JavaScript key
Go to developers.kakao.com, sign in, open My Application → Add an application. Then open the app → App Keys and copy the JavaScript key (not the REST or Admin key).
2
Switch the Kakao Map feature on
In the app: Kakao Map → Activation → ON. Without this, searches return nothing.
3
Register the web address you'll open this from
In the app: App Settings → Platform → Web, and add the site domain. Kakao only accepts requests from addresses you register here.
This page is open at:—
4
Paste the key and test it
Paste the JavaScript key below and press Test & save. The tool checks it against this exact address and turns on the Kakao panel if it works.
Put it on a link for everyone (recommended for sharing)

Don't send the file around — host it once and share the link. Colleagues just open it: no key, no install, and the Kakao panel works for them too.

  • Set the key once. Open this file in a text editor and put your JavaScript key in BAKED_KEY near the top of the script. It now ships inside the page, so nobody else pastes anything. A JavaScript key is domain-locked, so it's safe to expose as long as step 3 is done — it won't work from any other address.
  • GitHub Pages — works fully. Put this file (named index.html) in a repo, turn on Pages, and you get a link like https://you.github.io/repo/. Register https://you.github.io in step 3. The side-by-side satellite works because that's a real HTTPS address Kakao accepts.

Coordinates are WGS84, the system Google, Kakao and GPS share. The Google link uses ?q=lat,lng, which pins the exact point and sidesteps Korean-address search quality entirely. The Lite path reads Kakao's place page through a public relay, so it's best-effort — fine for spot-checks. Doing thousands of rows? A Kakao REST key in a script is the batch tool, not a browser.
